# RPA Planning Management

Web application that replaces the **RPA SV Summary** Excel workbook with a year-aware planning system covering revenue, projects, and resources.

| Layer        | Tech                                                   | Port |
| ------------ | ------------------------------------------------------ | ---- |
| **Frontend** | React 18 + react-router 6 + Tailwind 3 (Vite)          | 3000 |
| **Backend**  | Node.js 20 + Express 4 + JWT + bcrypt                  | 5000 |
| **Database** | PostgreSQL 13+ (external host)                          | 5432 |

The whole project is split into two npm workspaces (`backend/` and `frontend/`) plus a top-level orchestrator that runs both with `concurrently`.

## 2) Install

```bash
# from the project root
npm install                        # installs concurrently
npm run install:all                # installs backend + frontend
cp backend/.env.example backend/.env
```

## 3) Apply the database schema

```bash
psql -h 10.98.68.254 -U postgres -d rpa_planning -f backend/sql/schema.sql
```

The backend will also try to apply the schema automatically on first start (idempotent вЂ” uses `IF NOT EXISTS` everywhere). You can skip the manual `psql` step if your backend host can reach the DB.

## 4) Migrate data from the existing Excel

```bash
# Generate SQL + a mapping report from the workbook
node backend/scripts/migrateFromExcel.js path/to/RPA\ SV\ Summary\ -\ 2026.xlsx
# в†’ writes backend/sql/migration.sql and backend/sql/migration_report.md

# Apply it
psql -h 10.98.68.254 -U postgres -d rpa_planning -f backend/sql/migration.sql
```

Read `backend/sql/migration_report.md` afterwards вЂ” it lists every row that needed a synthetic project code, every customer alias the importer guessed, and the fields that the Excel didn't carry (notably outsource cost is always 0 in the source).

## 5) Run

```bash
# Both servers in one shell
npm run dev

# вЂ¦or separately
npm --prefix backend run dev      # http://localhost:5000
npm --prefix frontend run dev     # http://localhost:3000
```

Open <http://localhost:3000>. The app boots in **View-Only Guest mode** вЂ” every dashboard is visible. Click **Login** (top-right) with:

- **Username:** `superadmin`
- **Password:** `bsmrpa1234`

You'll be forced to change the password on first sign-in.

## Revenue recognition logic

All implemented in `backend/src/utils/revenue.js` and unit-friendly:

| Project Type             | % Recognize formula (year `Y`)                                                            |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| Subscription             | `days_of_contract_in_Y / total_contract_days`                                             |
| Perpetual License        | `1` if license `start_date.year == Y`, else `0`                                           |
| Software MA              | `days_of_contract_in_Y / total_contract_days`                                             |
| Service MA               | `days_of_contract_in_Y / total_contract_days`                                             |
| Implementation           | `progress_this_year - progress_last_year` (clipped to `[0,1]`)                            |
| Outsource вЂ” Man-Year     | `days_of_contract_in_Y / total_contract_days`                                             |
| Outsource вЂ” Man-Month    | `1` (sum of monthly entries for `Y` is taken as the recognized amount)                    |

`Recognize Revenue = revenue Г— % Recognize` and `Recognize Gross Margin = (revenue - cost) Г— % Recognize`.

The Summary page rolls up four buckets per the spec:

- **Pipeline License Revenue** вЂ” sum of recognize-GM for Subscription + Perpetual + SW MA where status = `Pipeline`.
- **Pipeline Service Revenue** вЂ” sum of recognize-revenue for Service MA + Outsource + Implementation where status = `Pipeline`.
- **Backlog & Win License Revenue** вЂ” same as above for `Win` / `Backlog`.
- **Backlog & Win Service Revenue** вЂ” same.
- **Target** = `headcount Г— revenue_per_headcount` (configured per year in **Admin в†’ Year Config**).

## Production deployment notes

- Set a strong `JWT_SECRET` in `.env`.
- Put both servers behind nginx/Caddy; proxy `/api/*` to backend, everything else to the built `frontend/dist`.
- Rotate the default superadmin password immediately on first login.
- Set up `pg_dump` backups for the `rpa_planning` database.
- For Gmail SMTP, use [App Passwords](https://myaccount.google.com/apppasswords); 2FA must be enabled on the sending account.
